[11303] 관리대장 누락
--------------------------------------------------
설명: 보안관리자용 기록관리 시스템에 등록되지 않은 호스트 또는 서비스

  □ 1. 전체 IP 자산에 대한 관리대장 작성 여부 확인
  □ 2. 관리대장과 실제 운영 자산의 일치성 검토
  □ 3. 미등록 자산 발견 시 즉시 등록 절차 수행
  □ 4. 주기적(월 1회) 자산 현황 점검 및 업데이트

조치방법: 모든 자산과 서비스에 대해 관리대장을 작성하고 주기적으로 검토해야 합니다.


[20501] 접근통제 미흡
--------------------------------------------------
설명: 익명 FTP, PostgreSQL 등 서비스에 기본 계정/익명 접근이 가능한 경우

  □ 1. FTP 익명 접속 비활성화 확인
  □ 2. 데이터베이스 기본 계정 비활성화 또는 패스워드 변경
  □ 3. SSH 루트 로그인 비활성화 확인
  □ 4. 서비스별 강력한 인증 정책 적용 확인

조치방법: 서비스 접근 시 인증체계를 적용하고 기본 계정 및 익명 접근을 제한해야 합니다.


[20502] SSH 약한 인증
--------------------------------------------------
설명: SSH 서비스에서 약한 패스워드 사용 또는 루트 로그인 허용

  □ 1. SSH 루트 로그인 비활성화 (PermitRootLogin no)
  □ 2. SSH 키 기반 인증 사용
  □ 3. SSH 접속 시도 제한 설정
  □ 4. SSH 기본 포트 변경 고려

조치방법: SSH 루트 로그인을 비활성화하고 키 기반 인증을 사용해야 합니다.


[20503] 기본포트 사용 및 취약한 서비스 운용
--------------------------------------------------
설명: Telnet, FTP, rlogin 등 보안성이 취약한 서비스가 기본 포트로 운용되는 경우

  □ 1. Telnet 서비스 비활성화 및 SSH로 대체
  □ 2. FTP 서비스 비활성화 및 SFTP/SCP로 대체
  □ 3. rlogin, rsh 등 r-command 서비스 비활성화
  □ 4. 필요시 VPN 내에서만 접근 가능하도록 구성

조치방법: 보안성이 낮은 서비스를 사용하지 않거나, VPN 내에서 사용하며 SSH 등 보안 대체 수단을 사용해야 합니다.


[30301] 관리대장 누락
--------------------------------------------------
설명: 연결된 장비가 관리대장에 포함되지 않음 (물리 네트워크 추적 기준)

  □ 1. 네트워크 연결 장비의 MAC 주소 수집
  □ 2. 물리적 네트워크 토폴로지 문서화
  □ 3. 미등록 네트워크 장비 탐지 및 등록
  □ 4. 네트워크 장비 관리대장 주기적 업데이트

조치방법: 물리적/논리적 연결 장비의 MAC 주소를 식별하고 자산 등록을 철저히 해야 합니다.


[30501] 불필요한 서비스 운영
--------------------------------------------------
설명: 업무와 무관한 불필요한 서비스가 실행 중인 경우

  □ 1. 실행 중인 모든 서비스 목록 작성
  □ 2. 업무 필요성에 따른 서비스 분류
  □ 3. 불필요한 서비스 중지 및 비활성화
  □ 4. 주기적 서비스 현황 점검

조치방법: 업무에 필요하지 않은 서비스는 중지하고 필요한 서비스만 운영해야 합니다.


[30601] SNMP 보안 설정 미흡
--------------------------------------------------
설명: SNMP 서비스에서 기본 커뮤니티 스트링 사용 또는 읽기/쓰기 권한 부여

  □ 1. SNMP 기본 커뮤니티 스트링(public, private) 변경
  □ 2. SNMP 읽기 전용 권한 설정
  □ 3. SNMP 접근 IP 제한 설정
  □ 4. SNMPv3 사용 검토

조치방법: SNMP 커뮤니티 스트링을 변경하고 읽기 전용으로 설정해야 합니다.


[30701] 웹 서버 보안 설정 미흡
--------------------------------------------------
설명: 웹 서버에서 디렉토리 리스팅, 불필요한 HTTP 메소드 허용 등

  □ 1. 디렉토리 브라우징 비활성화
  □ 2. 불필요한 HTTP 메소드(TRACE, DELETE 등) 비활성화
  □ 3. 에러 페이지에서 서버 정보 노출 방지
  □ 4. 보안 헤더 설정 (X-Frame-Options, X-XSS-Protection 등)

조치방법: 웹 서버 보안 설정을 강화하고 불필요한 기능을 비활성화해야 합니다.


[30802] 버전정보 노출
--------------------------------------------------
설명: 서비스에서 버전 정보가 노출될 경우 공격자가 취약점을 파악할 수 있음

  □ 1. 웹 서버 버전 정보 숨김 설정
  □ 2. SSH 배너 정보 최소화
  □ 3. FTP 환영 메시지에서 버전 정보 제거
  □ 4. 에러 메시지에서 시스템 정보 노출 방지

조치방법: 서비스 배너, 오류 메시지 등을 통해 버전 정보가 노출되지 않도록 설정합니다.


[40101] 패치 관리 미흡
--------------------------------------------------
설명: 시스템 및 애플리케이션의 보안 패치가 적용되지 않은 경우

  □ 1. 운영체제 최신 보안 패치 적용 여부 확인
  □ 2. 애플리케이션 보안 업데이트 적용 여부 확인
  □ 3. 패치 적용 전 테스트 절차 수립
  □ 4. 긴급 패치 적용 절차 수립

조치방법: 정기적인 보안 패치 적용 및 패치 관리 정책을 수립해야 합니다.
